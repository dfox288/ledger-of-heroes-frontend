**Date:** 2025-11-23
**Purpose:** Visual design specifications for new filtering capabilities
**Status:** ğŸ¨ Design Phase

## Design Principles

### 1. **Progressive Disclosure**
- Basic filters visible by default (search, most common filters)
- Advanced filters in collapsible section or "More Filters" button
- Keep filter bar clean and uncluttered

### 2. **Visual Hierarchy**
- Search bar most prominent (full width)
- Primary filters in row 2 (dropdowns/toggles)
- Active filter chips in row 3 (dismissible)
- Advanced filters collapsible below

### 3. **Consistency**
- Reuse existing filter UI patterns from current pages
- Maintain NuxtUI component styling (colors, spacing, borders)
- Keep filter behavior consistent across all entity pages

### 4. **Mobile-First**
- Filters stack vertically on mobile
- Collapsible filter panel to save screen space
- Filter button with badge showing active filter count

## Color Palette & Styling

### Filter States

#### Active Filter Chip Colors (by entity)
- **Spells:** Purple (`primary`)
- **Items:** Orange (`warning`)
- **Monsters:** Red (`error`)
- **Races:** Blue (`info`)
- **Classes:** Red (`error`)
- **Backgrounds:** Green (`success`)
- **Feats:** Orange (`warning`)

#### Component States
- **Active Toggle:** Entity color with white text
- **Inactive Toggle:** Neutral gray (`neutral`)
- **Hover State:** Slightly darker shade
- **Disabled State:** Light gray, 50% opacity

### Spacing & Layout
- Filter bar padding: `py-4 px-6`
- Filter row gap: `gap-3`
- Filter group spacing: `space-y-4`
- Active chip spacing: `gap-2`
- Mobile drawer padding: `p-6`

---

## Accessibility Considerations

### Keyboard Navigation
- All filter controls focusable with Tab key
- Enter/Space to toggle/select
- Escape to close dropdowns/modals
- Arrow keys to navigate within dropdowns

### Screen Readers
- Clear labels for all filter controls
- Announce filter state changes
- Announce active filter count
- Clear button includes "Clear all filters" label

### Visual Indicators
- Sufficient color contrast (WCAG AA minimum)
- Don't rely on color alone (use icons + text)
- Focus rings on interactive elements
- Loading states clearly indicated

---

## Animation & Interactions

### Transitions
- Filter panel expand/collapse: 200ms ease
- Dropdown open/close: 150ms ease
- Filter chip add/remove: 100ms ease
- Active filter count badge: pulse animation

### Micro-interactions
- Hover state: Gentle scale (1.02x)
- Click feedback: Brief scale down (0.98x)
- Filter applied: Success checkmark animation
- Clear filter: Fade out animation
